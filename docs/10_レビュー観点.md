# 10 レビュー観点

## 共通

- 命名が一貫しているか
- レイヤー責務が守られているか
- 公開リポジトリに不適切な情報が入っていないか
- `docs` と実装に差分がないか

## バックエンド

- `controller -> service -> dao -> mapper` の責務分離が守られているか
- SQL の配置が規約どおりか
- Mapper interface と Mapper XML の対応が取れているか
- DTO、DAO、Mapper、Model の役割が混在していないか
- 例外が適切な HTTP ステータスへ変換されているか

## フロントエンド

- 画面表示と API 通信の責務が分離されているか
- モック実装が残ったままになっていないか
- 通信失敗時の考慮が入っているか
